## Задание по докеру или симулятор сис админа

### Общая суть задания:
Развертывание backend сервера и прокси перед ним (В отдельных докер контейнерах). Прокси перенаправляет подключение на backend подсервер. Доступ к backend подсерверу возможен только через прокси.
Для выполнения задания потребуется докер, желательно система линукса для возможности ручного тестирования без контейнера.

### Краткая информация по софту, с которым предстоит работать:
Как и прокси, так и конечный сервер представляют из себя некий файл JAR именуемый ядром. Это ядро запускается как Java приложение.
При первом запуске конечного сервера будет сгенерирован файл `eula.txt`, где нужно заменить false на true (`eula=true`) - согласие с лицензионным соглашением.
После следующего запуска создается набор конфигурационных файлов (Обычно YML, TOML, PROPERTIES) и запускается сервер на настройках из этих файлов (Соответственно при повторных запусках будут использоваться настройки из этих файлов).

### Задача:
Требуется скачать с сайта PaperMC прокси Velocity (последняя версия и сборка) и backend-сервер Paper версии 1.16.5 последней сборки. На этом же сайте представлена подробная документация как запускать соответствующие ядра из-под разных систем.
Для нашей задачи требуется запустить оба сервера (прокси и backend) на разных докер контейнерах с ubuntu (последний LTS) на GraalVM.
В зависимости от желаемой оценки можно запустить либо на последней версии GraalVM, либо на минимально подходящей версии для каждого ядра соответственно (!). Подсказка: Воспользуйтесь утилитой, которая по JAR файлу определит версию, на которой он был собран.
Требуется использовать тома или монтировать внешнюю директорию для сохранения директорий world* при перезапуске в backend-подсервере (описать в отчете выбор и способ реализации).

##### В настройках backend-сервера выставить поля в файле server.properties:
- motd=backend сервер <ник тг> (Например, `motd=backend сервер @kiario`)
- online-mode=false
- query.port=25570
- server-port=25570
##### В настройках velocity-сервера выставить поле в файле velocity.toml:
- ping-passthrough = "all"
- В разделе `[servers]` указать адрес backend-сервера, доступный из докер контейнера.

Внутри докер контейнеров прокси будет работать на порте 25565, backend-сервер на порте 25570. Поскольку оба ядра выглядят как отдельные докер контейнеры нужно составить docker compose и запускать его.
**Важно:** Существуют уже готовые образы для развертывания таких серверов, в нашей задаче требуется непосредственно контейнер на ubuntu с ручной установкой

### Проверить, что задание выполнено корректно, можно двумя способами:
1. (Сложный, для фанатиков) Используя лицензионный клиент или клиент TLauncher версии 1.16.5, добавив сервер с IP "localhost:25565"
2. (Рекомендуется) Используя приложенный скрипт mc_motd_retriever.py
В обоих случаях результатом требуется увидеть свое имя или ник при пропинговке порта 25565 и увидеть превышение времени ожидания при пропинговке порта 25570 соответственно.
