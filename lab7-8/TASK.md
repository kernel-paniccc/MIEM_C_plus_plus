Лабораторная 7-8.
Цель лабораторной работы — реализация класса на языке C++ для управления связанным
списком блоков управления процессами (PCB). PCB (Process Control Block) — это структура данных,
используемая операционной системой для хранения информации о состоянии процесса. Эта
информация включает идентификатор процесса, его статус, счетчик команд, регистры CPU и
другие данные, необходимые для управления выполнением процесса.
Описание классов:
Структура PCB:

Атрибуты:
oprocessID: Идентификатор процесса (int).
oprocessName: Название процесса (std::string).
oprocessStatus: Статус процесса (std::string или enum, например, Running, Waiting,
Stopped).
ocommandCounter: Счетчик команд (int), указывающий на следующую исполняемую
команду.
ocpuRegisters: Массив или структура для хранения состояния регистров CPU (может
быть представлен, например, массивом int или специфической структурой).
Класс ListNode:


Атрибуты:
odata: Объект PCB.
onext: Указатель на следующий узел в списке (ListNode*).
Методы:
oКонструктор, инициализирующий data и next.
oДеструктор, если требуется.
Класс ProcessList:

Атрибуты:

head: Указатель на начало списка (ListNode*).
Методы:
oinsert(const PCB& newPCB): Вставляет новый PCB в список, упорядоченно по
идентификатору процесса. Возвращает true, если вставка успешна, и false в случае
дублирования идентификатора.
oremove(int pid): Удаляет PCB из списка по идентификатору. Возвращает true, если
удаление успешно, и false, если идентификатор не найден.
oprintList(): Выводит информацию о всех процессах в списке.
Необходимо предоставить файлы .cpp и соответствующие файлы .hВ функции main():
1. Создание списка процессов: Инициализация объекта класса ProcessList.
2. Добавление элементов: Вставка нескольких процессов в список.
3. Вывод списка: Печать списка после вставок для проверки корректности добавления.
4. Удаление элементов: Удаление одного или нескольких процессов из списка.
5. Вывод списка: Печать списка после удалений для демонстрации изменений.
6. Попытки неудачной операции: Демонстрация попыток вставки существующих и удаления
несуществующих элементов.
7. Финальный вывод списка: Печать конечного состояния списка.
